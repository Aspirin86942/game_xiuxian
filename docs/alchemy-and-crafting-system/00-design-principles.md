# 00. 设计决策准则

## 1. 目的

本文件记录炼丹、制作与丹药系统的设计判断标准，不记录当前实现细节。

## 2. 系统优化优先级

- 第一优先：把丹药做成可规划资源，而不是剧情彩蛋或纯随机补给。
- 第二优先：让战斗消耗、材料获取、炼制和突破准备形成完整闭环。
- 第三优先：在不破坏 `375x667` 单屏与旧存档兼容的前提下提升系统存在感。

## 3. 什么叫好抉择 / 好流程 / 好规则

- 玩家能说清自己为什么要去游历拿材料、为什么要炼哪种丹、为什么要把稀有资源留给突破。
- 丹药系统既服务前中期续航，也服务后期突破，但两类作用不互相吞噬。
- 自然回血只承担“最低限度不死档”的职责，不抢走回血丹的价值。
- 高境界突破仍保留门槛，但门槛来自准备程度而不是连续刷概率。
- 炼丹入口独立、规则明确、失败可见，不需要玩家猜测“这东西到底能不能炼”。
- 好抉择的标准是：现在的材料消耗、眼前的收益和未来的突破准备三者之间存在明确取舍。

## 4. 什么叫坏抉择 / 坏流程 / 坏规则

- 玩家只能靠升级回血，或战斗后被系统直接无代价抬回高血量。
- 丹药存在，但没有稳定来源、稳定配方和稳定使用场景。
- 稀有丹药与普通丹药效果重叠，导致玩家无法判断资源该花在哪里。
- 配方隐藏材料代价或隐藏锁定条件，玩家点击后才发现系统不让做。
- 把炼丹强行塞进现有 item action，导致扣料、发放产物和失败提示语言前后不一致。
- 坏抉择的标准是：无论玩家怎么选，材料去向、系统收益和未来路径都几乎没有区别。

## 5. 可接受的不透明

可以隐藏：

- 尚未解锁配方的完整文本和产物细节。
- 怪物掉落的精确随机率。
- 未来计划中的更高阶炼丹体系。

不可隐藏：

- 当前配方的材料需求。
- 成品丹药的直接效果与限制条件。
- 当前是否因战斗中、境界不足或已有药力而被系统阻断。
- 非战斗保底回血的速度、上限和适用范围。

## 6. 高风险线定义

- 任何会破坏 `state.inventory` 基本模型或旧存档兼容的改动都视为高风险线。
- 任何会让材料、成品或背包数量出现负数的结算都视为高风险线。
- 任何把自然回血和离线吐纳修为结算混成同一时间戳的做法都视为高风险线。
- 任何允许战斗中切到丹炉页或战斗中直接嗑药的改动都视为高风险线。

## 7. 系统一致性原则

- 一个改动至少应同时影响数据、状态、UI、测试四层，不允许只改其中一层再靠聊天补语义。
- 配方系统必须共享统一的“验证 -> 扣料 -> 发放 -> 提示 -> 存档”流程语言。
- 成品丹药继续沿用背包使用入口，不另开第二套使用界面。
- 高阶突破丹的价值必须来自“准备一次关键突破”，而不是“无限叠药冲上限”。
- 战斗续航改动必须同时审视战斗结算、丹药效果和自然回血阈值，不允许单点调参。
