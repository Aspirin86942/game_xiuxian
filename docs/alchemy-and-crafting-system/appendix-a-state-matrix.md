# 附录 A：状态矩阵

## 1. 配方生命周期矩阵

| 状态 | 是否在 `ALCHEMY_RECIPES` 中登记 | 丹炉页是否可见 | 当前是否可点击 | 是否会扣材料 | 是否会发放成品 | 说明 |
| --- | --- | --- | --- | --- | --- | --- |
| 未定义 | 否 | 否 | 否 | 否 | 否 | 仅存在于设计讨论中，不得进入 UI。 |
| 已定义但未解锁 | 是 | 是 | 否 | 否 | 否 | 需展示锁定原因，不展示成功按钮。 |
| 已解锁但材料不足 | 是 | 是 | 否 | 否 | 否 | 需展示缺料原因。 |
| 已解锁且材料足够 | 是 | 是 | 是 | 是 | 是 | 炼制成功后材料减少、成品增加。 |
| 炼制后库存为 0 的成本材料 | 是 | 是 | 视库存而定 | 是 | 是 | `changeItem()` 仍应删除归零键。 |
| 战斗中 | 是 | 否 | 否 | 否 | 否 | 当前规则禁止战斗内炼丹。 |

## 3. 自然回血矩阵

| 场景 | 是否结算自然回血 | 封顶规则 | 是否写入存档 | 说明 |
| --- | --- | --- | --- | --- |
| 页面常驻且非战斗 | 是 | `50% maxHp` | 是 | 通过定时器或时间差计算。 |
| 页面刷新后恢复 | 是 | `50% maxHp` | 是 | 依赖 `recovery.lastCheckedAt`。 |
| 自动吐纳离线修为结算 | 否 | 不适用 | 否 | 仍由 `offlineTraining` 负责。 |
| 战斗模态开启中 | 否 | 不适用 | 否 | 当前规则禁止战斗中恢复。 |

## 4. 存档兼容矩阵

| 场景 | 当前处理方式 | 风险点 | 兼容要求 |
| --- | --- | --- | --- |
| v5 存档缺 `recovery` | `mergeSave()` 回填默认恢复状态 | 中 | 必须可读，不得重置整个存档。 |
| v5 存档缺 `alchemy` 页签状态 | `ui.activeTab` 回填原默认页 | 低 | 不得因新增页签崩溃。 |
| 存在未知配方 ID | 当前无专门状态位 | 低 | 运行时应只按静态表渲染已知配方。 |
| 稀有突破丹数量归零 | `inventory` 删除该键 | 低 | 背包与丹炉页不应保留幽灵条目。 |
