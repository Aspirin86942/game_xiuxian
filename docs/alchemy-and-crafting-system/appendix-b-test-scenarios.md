# 附录 B：测试与设计验收场景

## 1. 当前自动化已覆盖场景

### 场景 1：现有成品丹药可直接使用

- Given：背包内已有 `juqidan`
- When：在 `tests/e2e/inventory.spec.js` 中点击使用按钮
- Then：修为文本刷新、存档中对应数量减少

### 场景 2：当前战斗结算会把气血写回固定比例

- Given：进入游历战斗
- When：`tests/e2e/adventure.spec.js` 跑完整战斗
- Then：断言仍依赖当前战斗后的固定 `hp` 结果

### 场景 3：旧存档缺较新字段时可回填

- Given：历史存档缺较新字段
- When：`tests/offline-smoke.js` 调用 `mergeSave()`
- Then：较新状态字段可被安全补齐

## 2. 本册要求新增的自动化场景

### 场景 4：配方成功后正确扣料并发放成品

- Given：背包材料足够，且配方已解锁
- When：执行 `craftRecipe(recipeId)`
- Then：材料减少、成品增加、库存不出现负数

### 场景 5：材料不足时阻断炼制

- Given：配方已解锁，但缺少至少一种材料
- When：执行 `craftRecipe(recipeId)`
- Then：返回明确错误，库存和成品状态都不变

### 场景 6：非战斗自然回血按间隔结算并封顶

- Given：玩家残血、当前非战斗、`recovery.lastCheckedAt` 可用
- When：执行 `resolveNaturalRecovery(nowMs)`
- Then：气血按间隔恢复，但最多到 `50% maxHp`

### 场景 7：高阶突破丹受境界限制与药力互斥限制

- Given：玩家持有高阶突破丹
- When：低境界尝试使用，或已有 `breakthroughBonus` 时再次使用
- Then：系统阻断并返回明确错误

### 场景 8：战斗结算不再沿用旧的固定回血比例

- Given：进入游历战斗并完成结算
- When：读取战斗后的 `state.playerStats.hp`
- Then：胜利时保留真实剩余血量，失败时落到新的保命血线

### 场景 9：战斗中不能切入丹炉页

- Given：战斗模态框已开启
- When：点击底部导航 `alchemy`
- Then：页面不切换，系统给出阻断或静默无效但状态不污染

### 场景 10：新增第 6 个底部导航按钮后移动端布局仍稳定

- Given：固定 `375x667` 视口
- When：进入主页面并切换各标签页
- Then：导航不换行、不遮挡、按钮可点击

## 3. 文档要求的设计审计场景

- 玩家能否一眼分辨材料、成品丹药、高阶突破丹三种对象。
- 自然回血是否只把玩家抬到“能继续走”的区间，而不是替代回血丹。
- 高阶突破丹是否只服务元婴及以上节点，而没有提前破坏前期成长节奏。
- 配方系统是否真的形成“材料来源 -> 炼制 -> 使用”的闭环，而不是单独多一个展示页。

## 4. 当前已知测试风险

- 当前仓库基线 `npm run test:smoke` 已有既有失败：`第 11 章选择失败: 不足：升仙令 x1`。
- 该失败与本册目标无直接关系，但会影响后续回归结果归因。
- 后续实现时必须明确区分“既有失败”与“新回归失败”。
