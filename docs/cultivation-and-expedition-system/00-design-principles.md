# 00. 设计决策准则

## 1. 目的

本文件记录修炼与游历主循环系统的设计判断标准，而不是当前实现细节。

## 2. 系统优化优先级

- 第一优先：`Correctness（正确性）` —— 主循环必须闭环自洽，不能再出现多个可重复修为入口互相打架。
- 第二优先：`Maintainability（可维护性）` —— 修炼、游历、战斗、突破与剧情推进必须有清晰主责，不再把核心节奏散在多个按钮与时机里。
- 第三优先：`Observability（可观测性）` —— 玩家必须看得懂资源从哪里来、花到哪里去、为什么现在能突破或不能突破。

## 3. 什么叫好抉择 / 好流程 / 好规则

- 好主循环：玩家能用一句话说清当前阶段该做什么，例如“先去游历赚灵石，再闭关把灵石炼成修为”。
- 好资源规则：同一种资源在主循环中只有一个主语义；灵石的主语义是“成长燃料 + 剧情/交易代价”，不是可随处直转修为的万能值。
- 好修炼规则：修炼是明确的资源消耗入口，而不是高频点击动作或离线自动累积。
- 好游历规则：游历是资源入口与风险入口，战斗只是游历事件之一，不应垄断整个游历页。
- 好突破规则：突破只在修为满时触发，结果只影响境界推进，不夹带额外隐藏扣费。
- 好剧情衔接：剧情主线继续由境界和既有章节条件推进，主循环只负责“把玩家送到能读下一段剧情的位置”。

## 4. 什么叫坏抉择 / 坏流程 / 坏规则

- 坏主循环：玩家最优动作重新退化成“反复点一个按钮”或“离线等时间自己涨修为”。
- 坏资源规则：同一种资源既能在背包里直接吸收，又能在修炼页消耗，还没有任何优先级说明。
- 坏游历规则：玩家点“游历”后每次都固定进战斗，导致事件池、资源采集、剧情线索都失去存在感。
- 坏失败规则：战斗失败同时扣修为、扣资源、卡剧情，形成过度惩罚。
- 坏单机边界：一边保留完整导出/导入，一边又假装提供可信平衡或反回档承诺。

## 5. 可接受的不透明

可以隐藏：

- 游历事件池内部的随机抽取细节。
- 战斗内每回合的轻微伤害波动。
- 幕后用于决定日志文案或浮字文案的格式细节。

不可隐藏：

- 每枚灵石能转化多少修为。
- 当前还差多少修为才能突破。
- 本次游历得到或失去了哪些资源。
- 导出 / 导入是否属于完整存档，以及项目是否承诺平衡可信性。

## 6. 高风险线定义

- 新增第二条可重复修为入口，属于高风险线。
- 让灵石重新在背包里直接吸收，属于高风险线。
- 让游历再次固定为“点一次必打一场”，属于高风险线。
- 破坏 `cultivation`、`maxCultivation`、`inventory.lingshi`、`realmIndex / stageIndex` 的稳定语义，属于高风险线。
- 在未声明兼容策略的前提下修改完整导出/导入边界，属于高风险线。

## 7. 系统一致性原则

- 一个主循环改动至少应同时影响 `state`、`UI`、`日志/提示`、`测试` 四层。
- 所有可重复成长必须走同一条语言：`游历产资源 -> 修炼消耗资源 -> 修为满 -> 突破 -> 剧情推进`。
- “失败”只允许作为可恢复的节奏损耗，不允许无说明叠成多重惩罚。
- “纯单机自由”意味着允许导出、导入、回档与改档，但必须停止宣称平衡可信边界。
