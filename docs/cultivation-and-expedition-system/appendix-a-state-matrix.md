# 附录 A：状态矩阵

## 1. 主循环状态矩阵

| 状态 | 是否可闭关 | 是否可突破 | 是否可游历 | 主要目标 | 说明 |
| --- | --- | --- | --- | --- | --- |
| 灵石不足且修为未满 | 否 | 否 | 是 | 去游历拿资源 | 主循环默认资源短缺态 |
| 灵石充足且修为未满 | 是 | 否 | 是 | 在游历与闭关之间做取舍 | 主循环核心决策态 |
| 修为已满 | 否 | 是 | 是 | 选择立即突破或先做别事 | 主按钮应切换为突破 |
| 战斗进行中 | 否 | 否 | 否 | 结算当前战斗 | 不允许重复进入游历 |
| 旧档不兼容 | 否 | 否 | 否 | 阻断并重置 | 仅适用于 `version < 6` |

## 2. 游历结果矩阵

| 事件类型 | 是否进入战斗 | 资源变化 | 气血变化 | 修为变化 | 玩家可见反馈 |
| --- | --- | --- | --- | --- | --- |
| `resource` | 否 | 灵石增加 | 无 | 无 | 灵石获得提示 + 日志 |
| `battle` 胜利 | 是 | 灵石增加 + 掉落 | 回到安全值 | 无 | 战斗日志 + 摘要 |
| `battle` 失败 | 是 | 灵石减少 | 回到安全值 | 无 | 战斗日志 + 损失提示 |
| `risk` | 否 | 灵石减少 | 扣少量最大气血比例 | 无 | 风险提示 + 日志 |
| `clue` | 否 | 通常无或少量 | 无 | 无 | 线索提示 / 剧情目标提示 |

## 3. 修炼状态矩阵

| 输入条件 | 实际行为 | 灵石是否变化 | 修为是否变化 | 结果 |
| --- | --- | --- | --- | --- |
| 修为未满且灵石充足 | 进入闭关 | 是 | 是 | 正常闭关 |
| 修为未满但灵石不足 | 阻断 | 否 | 否 | 显式失败提示 |
| 修为已满 | 转交突破 | 否 | 视突破结果 | 主按钮走突破语义 |
| 批量大于剩余可修次数 | 自动裁剪 | 按实际可修次数扣费 | 按实际可修次数结算 | 不允许溢出扣费 |

## 4. 存档兼容矩阵

| 场景 | 当前处理方式 | 风险点 | 兼容要求 |
| --- | --- | --- | --- |
| `version < 6` 的旧档 | 阻断并重置 | 丢失旧主循环进度 | 必须显式提示 |
| `version >= 6` 缺少次要字段 | `mergeSave()` 回填 | 中 | 不得崩溃 |
| 完整导出后导入同版存档 | 保留主状态 | 中 | 主状态应一致 |
| 导入异常 JSON | 阻断并提示错误 | 低 | 不污染当前状态 |
