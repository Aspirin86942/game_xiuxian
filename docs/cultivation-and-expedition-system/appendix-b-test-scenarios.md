# 附录 B：测试与设计验收场景

## 1. 当前自动化必须覆盖的场景

### 场景 1：灵石足够时闭关成功

- Given：背包中有足够 `lingshi`
- When：执行 `trainWithLingshi(state, "1")`
- Then：灵石减少、修为增加、修为不超过 `maxCultivation`

### 场景 2：灵石不足时闭关失败

- Given：背包中没有足够 `lingshi`
- When：执行闭关
- Then：返回显式失败，灵石与修为都不变

### 场景 3：修为已满时主按钮直接突破

- Given：`cultivation >= maxCultivation`
- When：点击主按钮
- Then：不再扣灵石，直接走突破

### 场景 4：资源事件只加灵石不加修为

- Given：游历命中 `resource`
- When：完成结算
- Then：`inventory.lingshi` 增加，`cultivation` 不变

### 场景 5：战斗胜利只给灵石和掉落

- Given：游历命中 `battle` 并胜利
- When：战斗完成
- Then：灵石和掉落增加，修为不变

### 场景 6：战斗失败不扣修为

- Given：游历命中 `battle` 并失败
- When：战斗完成
- Then：修为不变，灵石按规则减少，库存不为负

### 场景 7：旧档被阻断

- Given：`version = 5` 的历史存档
- When：加载或导入
- Then：显示不兼容提示并重置

### 场景 8：完整导出 / 导入可回放

- Given：`version = 6` 的当前存档
- When：导出后重新导入
- Then：境界、修为、灵石、剧情与关系字段保持一致

## 2. 端到端测试最低链路

### 场景 9：新档完成“游历 → 闭关 → 突破”

- Given：新档或低资源档
- When：先游历拿灵石，再闭关，再突破
- Then：境界推进且剧情目标更新

### 场景 10：设置页单机自由提示可见

- Given：打开设置页
- When：查看存档区域
- Then：能看到“完整导出 / 导入，纯单机自由，不保证平衡”的提示

### 场景 11：刷新后无离线收益弹层

- Given：完成一次正常游玩
- When：刷新页面
- Then：不出现离线收益弹层，也不会恢复自动吐纳

### 场景 12：375x667 视口下控件无遮挡

- Given：移动端固定视口
- When：打开修炼页与设置页
- Then：灵石摘要、闭关批次按钮、主按钮、底部导航都可见可点

## 3. 设计审计清单

- 玩家能否解释灵石是“闭关燃料”，而不是任意入口的万能成长值。
- 玩家能否在游历前理解“这次可能打架、可能拿资源、可能吃风险”。
- 玩家能否在设置页理解当前项目是单机自由，而不是可信竞赛环境。
- 任何失败路径是否都有显式提示，而不是点击后无反馈。
