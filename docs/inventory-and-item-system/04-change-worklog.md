# 04. 规则变更记录

## 1. 使用规则

本文件只记录储物、灵石消费与物品作用系统的规则变化，不记录普通代码重构或纯文案修订。

要求：

- 一次规则变化记一条，不把多个独立前提混成一段流水账。
- 必须写明为什么改、影响哪类物品、是否影响旧存档、何时应该回退。
- 若兼容影响或数值影响没有确切信息，明确写“需补验证”，不凭空补结论。

## 3. 设计前提

- `灵石必须至少有一条清晰消费路径`：当前为是；未来若再加入吸收路径，也必须保留“消费 vs 成长”的明确取舍。
- `持有型被动默认不依赖装备位`：当前为是；若未来要改，视为 A 级变更。
- `旧存档兼容优先于新机制炫技`：当前为是；所有破坏式改动都必须先写兼容策略。
- `一件物品必须能被解释成一种主语义`：当前为是；若新增复合语义物品，必须在规则书写清主次。

## 4. 回退条件

- 若新增规则导致旧存档无法读档，应回退。
- 若新增流程会让库存出现负数或扣费后继续走后续效果，应回退。
- 若新物品只能显示在文案里，却没有稳定的系统读取点，应回退。
- 若灵石新用途让剧情消费失去存在感，或使成长节奏明显失真，应回退并重新评估参数。

### `[2026-03-21] 新建储物、灵石消费与物品作用系统成熟分册`

#### 改动内容

- 新建 `inventory-and-item-system` 分册。
- 把“背包、灵石消费、可直接使用物品、持有型被动”收束成第二本 mature 分册。

#### 预期玩家体验 / 用户体验变化

- 本次仅新增规则书，不直接改变运行时行为。
- 后续实现阶段将能按统一契约补齐“灵石有花法”“特殊物品有系统身份”“持有型被动可解释”。
