# 附录 A：状态矩阵

## 1. 物品生命周期矩阵

| 状态 | 是否在 `inventory` 中有数量 | 背包是否可见 | 当前是否可点击 | 是否会影响数值 | 是否写入存档 | 说明 |
| --- | --- | --- | --- | --- | --- | --- |
| 未获得 | 否 | 否 | 否 | 否 | 否 | 仅存在于 `ITEMS` 定义，不进入运行时库存。 |
| 已获得但不可直接使用 | 是 | 是 | 否 | 视物品而定 | 是 | 当前大多数材料、剧情物都在这一态。 |
| 已获得且可直接使用 | 是 | 是 | 是 | 是 | 是 | 当前聚气丹、筑基丹、解毒散属于这一态。 |
| 已获得且持有即生效 | 是 | 是 | 否 | 是 | 是 | 当前飞剑、护身法器、曲魂属于这一态。 |
| 被剧情门槛读取 | 是 | 视背包而定 | 否 | 间接 | 是 | 通过 `requirements.items` 或 `costs` 读取。 |
| 被消耗到 0 | 否 | 否 | 否 | 否 | 否 | 当前 `changeItem()` 会删除该键。 |
| 纯剧情伏笔 | 可能否 | 可能否 | 否 | 否 | 视实现而定 | 若无稳定读取点，不应冒充正式物品分支。 |

## 2. 灵石行为矩阵

| 行为 | 当前是否已实现 | 当前入口 | 结算结果 | 玩家可见性 | 备注 |
| --- | --- | --- | --- | --- | --- |
| 剧情代价支付 | 是 | `choice.costs.lingshi` | 先扣库存，再结算 choice 效果 | 失败会报资源不足，成功会写日志 | 第 9、10 章已验证。 |
| 剧情奖励获得 | 是 | `choice.effects.items.lingshi` / 随机事件 / 战斗掉落 | 增加库存 | 日志或剧情文本可见 | 当前最常见来源之一。 |
| 背包直接使用 | 否 | 无 | 无 | UI 无按钮 | `lingshi.usable` 当前为 `false`。 |
| 主动吸收转化 | 否（目标态预留） | 无 | 无 | 当前无提示 | 本册只建立规则位，不宣称已实现。 |
| 商店/交易支付 | 否（系统级） | 无独立面板 | 无 | 当前无入口 | 仅在剧情 choice 语义中存在“花灵石办事”。 |

## 3. 存档兼容矩阵

| 场景 | 当前处理方式 | 风险点 | 兼容要求 |
| --- | --- | --- | --- |
| 旧存档缺 `inventory` | `mergeSave()` 回填为 `{}` | 低 | 必须继续可读。 |
| 旧存档缺较新 UI 字段 | `mergeSave()` 回填默认 UI 状态 | 中 | 不得因新增背包按钮而读档崩溃。 |
| 旧存档已有未知物品键 | 当前无专门清洗 | 中 | 未来若清洗未知键，必须记录兼容策略。 |
| 库存数量为 0 | 当前删除键 | 低 | UI 不应显示残留空条目。 |
| 库存数量欲扣成负数 | `changeItem()` 返回 `false` | 高 | 后续效果不得继续执行。 |
