# 02. 参数与公式

## 1. 说明

本文件记录理想化叙事决策系统 v2 的目标参数、阈值、档位与公式。

重要说明：

- 本表中的 `当前值` 指“当前规则书的目标规格”，不是“当前代码已实装值”。
- 当前代码真实情况请回看 `01-current-behavior.md`。
- 若未来实施时决定调整某项目标规格，必须同步更新 `03-implementation-contract.md` 与 `04-change-worklog.md`。

## 2. 核心阈值

| 参数名 | 当前值 | 作用 | 调大后的影响 | 调小后的影响 | 风险 | 允许调整范围 |
| --- | --- | --- | --- | --- | --- | --- |
| 选前风险揭示档位 | 3 档 | 把 choice 的风险语义统一为 `稳妥 / 有压 / 涉险` 三档 | 风险表达更细，但文案和 UI 负担会上升 | 风险语义过粗，玩家更难理解承诺差异 | 档位太多会接近透题，档位太少会重新黑箱化 | 2~4 档 |
| 单次可见分支影响条数 | 1 条 / choice | 保证一次选择只留下 1 条可回看的可见影响 | 可见信息更多，但会重新堆出重复卡 | 可读性更强，但若降为 0 会失去选后解释 | 超过 1 条会迅速回到旧多卡冗余 | 固定为 1 |
| 分支影响标题长度 | 4~8 字 | 保证标题像主题名而不是说明句 | 意象更完整，但标题更易拖沓 | 更利落，但区分度可能不足 | 超过 8 字容易变成摘要句 | 4~10 字 |
| 分支影响正文长度 | 1~2 句 | 保证可回味但不啰嗦 | 氛围更浓，但阅读阻力上升 | 更利落，但承接感会变薄 | 超过 2 句会变成系统说明段落 | 1~2 句 |
| 隐藏承接兑现窗口 | 2~5 个章节单元 | 让后果延迟但不失联 | 余韵更长，但更难回指原始因果 | 承接更直接，但历史厚度变弱 | 窗口太长会让玩家忘记来源 | 1~6 个章节单元 |
| 分支历史显示上限 | 64 条 | 允许单局历史尽量完整保留 | 历史更完整，但列表更长 | UI 更轻，但历史回看会断层 | 上限太低会削弱“你之前选了什么”的体验 | 24~128 条 |
| 分支记忆主维度数 | 5 维 | 约束长期状态至少覆盖 `推进 / 路线 / 关系 / 压力 / 终局` | 表达更细，但作者负担增大 | 过少会退化成单一分值系统 | 维度失控会回到“状态海” | 4~6 维 |
| 失败压力危险区间 | 4 档 | 把失败压力拆成 `安全 / 紧绷 / 濒危 / 失控` | 可管理性更高，但实现更复杂 | 压力解释更简单，但管理空间缩小 | 档位过少会失去压线玩法 | 3~5 档 |
| 伪分支最低门槛 | 1 条可见分支影响 + 1 次分支记忆写入 + 1 个后续读取点 | 防止选择只改文案不改系统 | 可迫使内容更扎实 | 门槛过低时伪分支大量进入系统 | 门槛过高会压缩内容产能 | 不低于该值 |
| 终局种子回指深度 | 最近 2~4 次关键 choice | 保证终局可解释性 | 终局解释更厚实 | 终局解释更短促 | 过深会导致说明冗长 | 1~5 次 |
| 终局卷主终局数 | 5 条 | 保证第五卷把长期路线、关系与旧账收成稳定终局谱系 | 分流更细，但作者与测试负担上升 | 分流更粗，终局会重新退化成临时菜单 | 少于 5 条会丢失 issue #12 的终局语义 | 固定为 5 |
| 异常失败终局入口 | 1 条独立入口 | 保证异常失败与正常主终局分开结算 | 失败边界更清楚 | 失败与正常终局更易混写 | 若与主终局混在同一菜单，会削弱失败可解释性 | 固定为 1 |

## 3. 路径参数组

### 3.1 决策输入层

- 每个 choice 至少声明：
  - 一个主承诺类型
  - 一个可见成本或机会成本
  - 一个风险语义档位
- 若缺少其中任一项，该 choice 不应进入成熟内容。

### 3.2 后果输出层

- 每个成熟 choice 至少落地：
  - 一条可见分支影响
  - 一项分支记忆写入
  - 一个隐藏承接读取点或终局种子
- 高风险 choice 还必须让失败压力变化可被玩家识别。

### 3.3 分支记忆层

- 目标态不强制沿用当前字段名，但要求语义上至少保留：
  - 主线推进记忆
  - 路线人格权重
  - 关系承接
  - 失败压力
  - 终局种子

## 4. 公式

### 4.1 揭示完整度公式

```text
揭示完整度 = 可见承诺 + 可见成本/机会成本 + 可见风险语义
合格条件：三项至少满足两项，且风险语义在关键转折时不得缺失
```

### 4.2 后果落地公式

```text
后果落地 = 单条分支影响 + 分支记忆写入 + 隐藏承接注册
合格条件：三项至少满足两项，且单条分支影响必须可被玩家感知
```

### 4.3 伪分支判定公式

```text
若不同 choice 只共享同一承诺标签与风险标签，且复用同一分支影响标题 / 正文，则判定为伪分支
若 choice 没有后续读取点，或隐藏承接与终局回指完全相同，也判定为伪分支
```

### 4.4 失败压力分层公式

```text
失败压力分层 = 安全 -> 紧绷 -> 濒危 -> 失控
终局触发应来自“长期进入失控”或“关键抉择将既有压力推过失控线”，而不是随机抽签
```

### 4.5 终局解释公式

```text
终局解释 = 最近关键分支影响链 + 压力累计路径 + 最终触发 choice
目标：玩家能从终局描述中回溯自己为何走到这里
```

### 4.6 终局卷分流公式

```text
终局卷分流 = 异常失败终局优先判定 -> 主终局候选筛选 -> 玩家只在符合条件的主终局中选择 -> 写入 ending
目标：异常失败终局与正常主终局不共用同一候选菜单
```

### 4.7 终局卷章节压缩公式

```text
终局卷章序 = opening -> escalation -> bonding -> closure -> ending
目标：终局卷允许压缩为 5 章，但仍必须完整覆盖“立题、清账、认人、收束、落终局”
```

## 5. 观测指标

### 5.1 玩家可解释性指标

- 玩家是否能说出本次 choice 的承诺类型。
- 玩家是否能从分支影响标题与正文里说出这一步留下了什么。
- 玩家是否能说出当前失败压力是在上升、回稳还是已接近失控。

### 5.2 路线差异指标

- 三条主路线是否具有不同的风险姿态与关系代价。
- 同承诺类型的不同 choice 是否拥有不同的分支影响主题和正文。
- 是否存在明显严格优势解。

### 5.3 失败可管理性指标

- 玩家是否能识别自己正处于哪个失败压力区间。
- 高风险路线是否存在压线而非必死的通关空间。
- 终局说明是否能回指关键分支影响链。

### 5.4 内容扩写指标

- 作者是否能在不翻代码的前提下完成 choice 接入。
- 新章节是否能被附录检查项快速审计。
- 新写入状态是否能在后续找到明确读取点。

## 6. 联动参数

- 风险揭示档位 与 分支影响正文语气 强联动。
- 分支历史显示上限 与 剧情页可读性 强联动。
- 隐藏承接兑现窗口 与 终局种子回指深度 强联动。
- 分支记忆主维度数 与 内容作者负担 强联动。
- 失败压力危险区间 与 高风险路线压线空间 强联动。
- 终局卷主终局数 与 旧情、旧账、路线分流可解释性 强联动。
- 异常失败终局入口 与 失败压力分层公式 强联动。

## 7. 风险说明

- 最大风险不是“参数不够精确”，而是目标态参数与 `01` 的现状偏差失联，导致实施者误以为系统已经具备这些能力。
- 若继续把 `即时结果 / 长期提示 / 延迟余波` 都堆进可见层，系统会重新退化成“信息看似更多、其实重复更高”的旧多卡结构。
- 若分支影响只剩承诺标签与风险标签，没有独立主题和正文，系统会退化成计数器历史，而不是叙事历史。
- 若过度依赖兜底生成器，不补手写 `branchImpact`，内容会迅速模板化。
- 若先实现风险语义、不实现失败压力分层，系统会变成“看起来很诚实，实际上仍然难管理”的伪升级。
