# 04. 规则变更记录

## 1. 使用规则

本文件只记录叙事决策系统的规则级变化，不记录普通措辞润色或无意义的文件搬运。

每次记录至少写清：

- 改动内容
- 改动原因
- 预期体验变化
- 设计前提是否改变
- 是否影响旧状态 / 旧存档 / 旧客户端
- 是否需要同步更新测试
- 回退条件

若一次改动同时影响风险揭示、失败压力和内容接入门槛，必须拆开描述，不得写成一句“体验优化”。

### `[2026-03-22] 第一卷支线卷末收口接入运行时`

#### 改动内容

- 在第一卷出口对应的运行时章节里，新增对 `old_medicine_ledger` 与 `apothecary_boy_echo` 的主线读取。
- 第一卷第 `10/11` 章现在会根据既有 flags / relations 解释：
  - 墨府旧账是交回活人、按住名字，还是已通过承诺与补偿收住。
  - 曲魂 / 药童残影是被超度、被封存、被强带，还是被带成以后继续追问的旧案。
- 不改存档结构，不重编号 `storyProgress`，继续沿用旧 `0~11` 兼容层与第一卷 8 章显示映射。

#### 改动原因

- 规则书和元数据已经定义了第一卷卷末必须收口，但运行时主线此前还没有把两条支线真正解释出来。
- 如果只保留 `followupHook`、支线结果旗标和远期回响，玩家在第一卷离卷时仍会觉得“做过支线，但主线没认账”。

#### 预期玩家体验 / 用户体验变化

- 玩家到达第一卷出口时，能在主线正文里直接看见墨府旧账与药童残影的卷末意义。
- 第一卷离卷理由更完整，不再只靠支线卡或远期回响替代收尾。

#### 设计前提是否改变

- `选前揭示但不过度透题`：否
- `失败压力可管理而非随机暴毙`：否
- `choice 必须留下至少两层系统痕迹`：否
- `终局必须回指关键承诺链`：否

#### 是否影响旧状态 / 旧存档 / 旧客户端

- 否
- 影响点：
  - 旧存档进入第一卷卷末区时，剧情页会读到新增的主线解释文案。
  - 缺少相关 flags 的旧档会走中性收口文案，不会空白。

#### 是否需要同步更新测试

- 是

### `[2026-03-22] 第一卷卷结构与卷末回收规范入册`

#### 改动内容

- 将本册从“只约束单次 choice”扩展为“同时约束卷结构与章节角色”的规则书。
- 新增第一卷《七玄门风云》8 章卷结构附录，并固定旧 `0~11` 章的目标映射。
- 新增 `volumeId / volumeRole / chapterGoal / chapterConflict / closureWrites / nextReads` 作为主线章节目标态接口。
- 明确旧 `8 墨府旧事`、`9 曲魂初现` 不再作为并列主线尾章，而应转入卷内旧账回收或支线回收。

#### 预期玩家体验 / 用户体验变化

- 未来第一卷会更像完整开篇卷，而不是七玄门、墨府、太南小会、升仙令一路平铺到底。
- 玩家打完第一卷后，应能更清楚地理解本卷起点、反转、高潮与出口。

- `选前揭示但不过度透题`：否

### `[2026-03-21] 可见反馈改版为单条分支影响`

#### 改动内容

- 将剧情页右侧可见反馈收口为两层结构：
  - 顶部独立显示 `失败压力`
  - 下方仅显示累计 `分支影响`
- 停止把 `即时结果`、`长期提示`、`pendingEchoes`、旧 delayed echo 作为独立卡片并列推入剧情页。
- 将 `decisionHistory` 升级为剧情页唯一优先可见数据源，并补齐 `branchImpactTitle / branchImpactDetail`。
- 将终局回顾改为优先回指分支影响主题与正文。
- 保留 `pendingEchoes`、`immediateResult`、`longTermHint` 作为兼容输入或隐藏承接材料，但不再视为主显示契约。

#### 改动原因

- 旧多卡结构会把同一选择拆成多个同义说明，造成回响重复、阅读噪音和文案模板化。
- 用户明确要求“一次选择 = 一条分支影响”，并要求主题与正文更生动、可回味，而不是系统说明句。
- 若继续以 `pendingEchoes` 直接堆叠可见卡片，历史只会越来越像日志，不像分支记忆。

#### 预期玩家体验 / 用户体验变化

- 玩家选后只看到一条更完整的分支影响，而不是多张相互复述的卡片。
- 玩家回看历史时，能从主题、正文和元信息中识别“自己之前选了什么”。
- 失败压力继续保持独立可见，不与分支影响混写。
- 延迟承接仍会留在后面的路上，但不再通过可见多卡提前剧透或重复说明。

#### 设计前提是否改变

- `选前揭示但不过度透题`：否
- `失败压力可管理而非随机暴毙`：否
- `choice 必须留下至少两层系统痕迹`：否
- `终局必须回指关键承诺链`：否

#### 是否影响旧状态 / 旧存档 / 旧客户端

- 是
- 若是，影响点：
  - 旧的剧情页可见反馈结构被废弃
  - 旧历史若缺少 `branchImpact` 字段，需要走兼容兜底生成器
  - `pendingEchoes` 继续保留，但只作为兼容遗留数据

#### 是否需要同步更新测试

- 是
- 若是，需要更新：
  - 文档校验：是，需重跑单册与系列校验
  - smoke：是，需覆盖“单条分支影响、同承诺不同主题、旧历史兼容兜底”
  - e2e：是，需覆盖“失败压力顶置 + 单条分支影响卡”结构

#### 回退条件

- 若一次 choice 仍会在剧情页生成 2 条以上同义可见卡片
- 若 `echo-list` 再次出现 `即时结果` 或 `长期提示` 卡
- 若同承诺类型的不同 choice 大量复用相同标题或正文模板
- 若终局页无法回指新的分支影响链
- 若旧历史在兼容回退后出现空白卡或重复堆叠

#### 需要同步更新的文档

- `README.md`
- `01-current-behavior.md`
- `02-parameters-and-formulas.md`
- `03-implementation-contract.md`
- `05-content-authoring-interface.md`
- `appendix-a-state-matrix.md`
- `appendix-b-test-scenarios.md`
