# 04. 规则变更记录

## 1. 使用规则

本文件只记录叙事决策系统的规则级变化，不记录普通措辞润色或无意义的文件搬运。

每次记录至少写清：

- 改动内容
- 改动原因
- 预期体验变化
- 设计前提是否改变
- 是否影响旧状态 / 旧存档 / 旧客户端
- 是否需要同步更新测试
- 回退条件

若一次改动同时影响风险揭示、失败压力和内容接入门槛，必须拆开描述，不得写成一句“体验优化”。

## 3. 记录条目

### `[2026-03-21] 可见反馈改版为单条分支影响`

#### 改动内容

- 将剧情页右侧可见反馈收口为两层结构：
  - 顶部独立显示 `失败压力`
  - 下方仅显示累计 `分支影响`
- 停止把 `即时结果`、`长期提示`、`pendingEchoes`、旧 delayed echo 作为独立卡片并列推入剧情页。
- 将 `decisionHistory` 升级为剧情页唯一优先可见数据源，并补齐 `branchImpactTitle / branchImpactDetail`。
- 将终局回顾改为优先回指分支影响主题与正文。
- 保留 `pendingEchoes`、`immediateResult`、`longTermHint` 作为兼容输入或隐藏承接材料，但不再视为主显示契约。

#### 改动原因

- 旧多卡结构会把同一选择拆成多个同义说明，造成回响重复、阅读噪音和文案模板化。
- 用户明确要求“一次选择 = 一条分支影响”，并要求主题与正文更生动、可回味，而不是系统说明句。
- 若继续以 `pendingEchoes` 直接堆叠可见卡片，历史只会越来越像日志，不像分支记忆。

#### 预期玩家体验 / 用户体验变化

- 玩家选后只看到一条更完整的分支影响，而不是多张相互复述的卡片。
- 玩家回看历史时，能从主题、正文和元信息中识别“自己之前选了什么”。
- 失败压力继续保持独立可见，不与分支影响混写。
- 延迟承接仍会留在后面的路上，但不再通过可见多卡提前剧透或重复说明。

#### 设计前提是否改变

- `选前揭示但不过度透题`：否
- `失败压力可管理而非随机暴毙`：否
- `choice 必须留下至少两层系统痕迹`：否
- `终局必须回指关键承诺链`：否

#### 是否影响旧状态 / 旧存档 / 旧客户端

- 是
- 若是，影响点：
  - 旧的剧情页可见反馈结构被废弃
  - 旧历史若缺少 `branchImpact` 字段，需要走兼容兜底生成器
  - `pendingEchoes` 继续保留，但只作为兼容遗留数据

#### 是否需要同步更新测试

- 是
- 若是，需要更新：
  - 文档校验：是，需重跑单册与系列校验
  - smoke：是，需覆盖“单条分支影响、同承诺不同主题、旧历史兼容兜底”
  - e2e：是，需覆盖“失败压力顶置 + 单条分支影响卡”结构

#### 回退条件

- 若一次 choice 仍会在剧情页生成 2 条以上同义可见卡片
- 若 `echo-list` 再次出现 `即时结果` 或 `长期提示` 卡
- 若同承诺类型的不同 choice 大量复用相同标题或正文模板
- 若终局页无法回指新的分支影响链
- 若旧历史在兼容回退后出现空白卡或重复堆叠

#### 需要同步更新的文档

- `03-implementation-contract.md`
- `05-content-authoring-interface.md`

### `[2026-03-21] 理想化叙事决策系统规则书 v2 改写`

#### 改动内容

- 将 `narrative-decision-system` 从“现状事实书”改写为“理想目标态规则书”。
- 保留 `01-current-behavior.md`，但压缩为最小现状基线与关键偏差清单。
- 重写参数、实施契约、作者接口与附录，使其围绕目标态接口而不是当前字段堆叠。
- 同步更新系列索引定位。

#### 改动原因

- 原规则书更适合记录当前代码行为，不足以作为未来设计与实现的统一契约。
- 当前系统缺少对理想 choice、失败压力、延迟承接、作者接口的统一定义，继续沿用事实书会限制演进。

#### 预期玩家体验 / 用户体验变化

- 本次不直接改变玩家体验。
- 预期为后续实现提供统一目标态，从而避免新增内容继续沿用旧式弱分支和弱解释结构。

#### 设计前提是否改变

- `选前揭示但不过度透题`：是
- `失败压力可管理而非随机暴毙`：是
- `choice 必须留下至少两层系统痕迹`：是
- `终局必须回指关键承诺链`：是

#### 是否影响旧状态 / 旧存档 / 旧客户端

- 否
- 本次只改规则书，不改运行时代码。

#### 是否需要同步更新测试

- 若 `02`、`03`、`05` 之间出现术语漂移，导致同一概念跨文件含义不一致
