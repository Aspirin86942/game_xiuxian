# 04. 规则变更记录

## 1. 使用规则

本文件只记录叙事决策系统的规则级变化，不记录普通措辞润色或无意义的文件搬运。

每次记录至少写清：

- 改动内容
- 改动原因
- 预期体验变化
- 设计前提是否改变
- 是否影响旧状态 / 旧存档 / 旧客户端
- 是否需要同步更新测试
- 回退条件

若一次改动同时影响风险揭示、失败压力和内容接入门槛，必须拆开描述，不得写成一句“体验优化”。

## 3. 记录条目

### `[2026-03-21] 理想化叙事决策系统 v2 运行时代码全量落地`

#### 改动内容

- 在 `game-core.js` 落地 v2 状态层：
  - `version = 5`
  - `decisionHistory`
  - `pendingEchoes`
  - `endingSeeds`
  - `storyConsequences.pressureTier / pressureTrend`
- 在 `story-data.js` 对主线章节与小境界事件的全部 choice 做统一归一化，补齐承诺、风险、即时结果、长期提示、延迟回响与终局种子
- 在 `game.js / index.html / style.css` 落地剧情页 v2 揭示契约：
  - 选前展示承诺类型、可见代价、风险语义、不可执行原因
  - 选后固定展示“即时结果 → 长期提示 → 延迟回响”
  - 常驻显示压力档位与趋势
  - 终局页显示关键承诺链回指
- 在加载与导入流程中显式拒绝旧版存档
- 同步更新 smoke 与 e2e，覆盖压力四档、延迟回响窗口、终局回指与旧档导入阻断

#### 改动原因

- 规则书 v2 已定义完整的四层闭环，如果继续沿用旧的 `tradeoff + recentChoiceEcho` 过渡模型，会导致内容、UI 和测试继续分叉
- 用户已明确选择“一次性全量落地”与“旧档直接作废”，因此本次直接切换到 `version = 5`

#### 预期玩家体验 / 用户体验变化

- 玩家在选前就能知道自己承诺了什么、要付什么代价、正在面对什么风险
- 玩家在选后会先看到即时结果，再看到长期提示，延迟回响只在窗口内出现
- 失败压力由可见档位与趋势承接，不再以精确数值直接暴露
- 旧版存档不会再被静默兼容或污染当前状态

#### 设计前提是否改变

- `选前揭示但不过度透题`：否
- `失败压力可管理而非随机暴毙`：否
- `choice 必须留下至少两层系统痕迹`：否
- `终局必须回指关键承诺链`：否

#### 是否影响旧状态 / 旧存档 / 旧客户端

- 是
- 若是，影响点：
  - `version < 5` 的旧档不再兼容
  - 旧档加载时会重置为新档
  - 旧档导入时会直接阻断

#### 是否需要同步更新测试

- 是
- 若是，需要更新：
  - 文档校验：是，需重跑单册与系列校验
  - smoke：是，需覆盖 v2 状态写入、压力分层、延迟回响窗口、终局回指
  - e2e：是，需覆盖选前揭示、选后顺序、旧档导入阻断、终局承诺链

#### 回退条件

- 若 choice 仍暴露精确 `battleWill / tribulation` 数值
- 若延迟回响在未到窗口前就提前回退为旧逻辑
- 若高风险路线仍存在单次黑箱暴毙
- 若终局页无法回指最近 2~4 条关键承诺链
- 若旧版存档还能静默覆盖当前状态

#### 需要同步更新的文档

- `01-current-behavior.md`
- `04-change-worklog.md`
