# 05. 内容接入接口规范

## 1. 目的

本文件定义叙事决策系统 v2 的内容作者接口，确保卷、章节、choices、分支影响、卷末收束和跨卷读取点都能以统一方式接入系统。

内容接入不是纯写作任务，而是“叙事资产 + 系统接口”的联合设计。

## 2. 一卷主线最少要定义什么

一个合格的主线卷至少要定义：

- `volumeId`
- `volumeTitle`
- `volumePromise`
- `volumeExit`
- `maxForwardHooks`
- `核心章节列表`

每卷还必须明确：

- 本卷最核心的冲突是什么。
- 哪一章承担反转。
- 哪一章承担高潮。
- 哪一章负责清算代价。
- 哪些旧账必须卷内解决。
- 哪些内容允许带着读取点跨卷。

## 3. 一个主线章节最少要定义什么

一个合格的主线章节至少要定义：

- `volumeId`
- `volumeRole`
- `chapterGoal`
- `chapterConflict`
- `章节摘要`
- `beats`
- `choices`
- `closureWrites`
- `nextReads`

说明：

- `closureWrites` 回答“这一章替本卷收住了什么”。
- `nextReads` 回答“后面谁会来认这件事”。
- 若一章只写摘要和 choices，不写 `closureWrites / nextReads`，默认视为未完成资产。

## 4. 一个合格 choice 至少要有哪三层差异

### 4.1 表层差异：决策承诺

- 玩家必须能从按钮文案与上下文理解：这次到底在承诺什么。
- 承诺必须可区分，例如偏保全、偏献祭、偏试探、偏决裂、偏夺取。

### 4.2 中层差异：可见成本 / 风险语义

- 玩家至少要看到一种成本或机会成本。
- 玩家至少要看到一种风险语义，不要求精确值，但要知道自己是否在涉险。

### 4.3 深层差异：分支影响与卷内承接

- choice 至少命中以下三项中的两项：
  - 一条显式 `branchImpact`
  - 一项分支记忆写入
  - 一项 `closureWrites` 或 `nextReads`

高风险 choice 还必须说明失败压力为何会变化。

## 5. 何时必须写显式 `tradeoff`

以下情况必须显式声明 `tradeoff`：

- 关键剧情转折
- 逆路线默认气质的选择
- 明确承担高风险高收益语义的 choice
- 会显著推升失败压力或终局种子的 choice
- 会改变卷末出口解释的 choice

## 6. 何时只允许走默认 `tradeoff`

- 常规承接章节中的普通分流
- 不承担世界观反转的日常 choice
- 只是延续既有路线人格的稳定 choice

但即便走默认 `tradeoff`，成熟内容也仍需手写 `branchImpact`。

## 7. 什么是伪分支

以下情况默认属于伪分支：

- 文案不同，但 `closureWrites / nextReads` 完全相同。
- 只有数值差，没有卷内代价、关系、路线或读取点差异。
- 只留下隐藏状态，不留下可回看的 `branchImpact`。
- 需要作者自己记住“以后再补”，但规则书里没有登记读取点。
- 明明属于卷末清账，却继续伪装成下卷问题。

## 8. 第一卷作者接口默认

第一卷《七玄门风云》固定按 8 章接入，详见 `appendix-c-volume-one-structure.md`。

第一卷作者在接入时必须遵守：

1. 卷末最多只保留 3 个跨卷读取点。
2. 旧 `8 墨府旧事` 和 `9 曲魂初现` 不再以并列主线尾章接入。
3. 与墨府、药童、曲魂相关的内容，必须写成：
   - 卷内旧账回收
   - 支线回收
   - 或明确跨卷读取点
4. `10 太南小会` 与 `11 升仙令` 只能共同服务卷末出口，不能继续各自拉出新的卷末尾巴。

## 8.1 第二卷作者接口默认

第二卷《初踏修仙路》固定按 8 章接入，详见 `appendix-d-volume-two-structure.md`。

第二卷作者在接入时必须遵守：

1. 卷末必须止于 `禁地前夜 / 卷末收束`，不提前展开 `血色禁地 / 情债与筑基 / 李化元门下` 正文。
2. 第二卷卷末只允许保留以下四类读取点：
   - `forbidden_ground_entry`
   - `foundation_prep_ready`
   - `mortal_debt_carryover`
   - `sect_identity_locked`
3. 与散修交易、升仙令、黄枫谷立足、百药园积累有关的内容，必须明确写成第二卷内的成长闭环，而不是零散过场。
4. “凡俗旧债未清”这一章必须继续读取墨彩环与墨府旧因果，但不得把墨府线重开成独立长主线。
5. `宗门人际与禁地前夜` 与 `卷末收束` 必须把第三卷入口交待清楚，不能用“下章可玩”替代出口解释。

## 8.2 第三卷作者接口默认

第三卷《魔道入侵》固定按 8 章接入，详见 `appendix-e-volume-three-structure.md`。

第三卷作者在接入时必须遵守：

1. 第三卷起点固定为 `14 血色禁地`；issue #10 中“血色禁地前夜”的危险感、资源压力和试炼诱因只能吸收到第 1 章语义里，不得回退第二卷已锁定边界。
2. `18_nangong_return` 必须写成第三卷第 6 章 `fallout`，明确承担大战后的关系落点、亏欠处理和第一阶段锁点，不允许继续只写成情绪奖励回响。
3. `16_feiyu_return` 只能作为卷内插章读取“凡心未死”的旧友回声，不得承担第三卷核心反转、高潮、卷末出口或跨卷主解释。
4. 第三卷卷末只允许保留以下四类读取点：
   - `star_sea_entry`
   - `nangong_bond_stage_one`
   - `war_route_locked`
   - `postwar_resource_pressure`
5. `20 再别天南` 必须解释玩家为什么要离开天南进入下一卷，不能用“地图更大了”或“下章可玩”替代出口说明。

## 9. 作者最小接入清单

新增章节或 choices 前，至少检查：

1. 本章在卷内承担什么角色。
2. 本章最核心的冲突是什么。
3. 本章会替本卷收住什么。
4. 本章会把哪些内容交给后续读取。
5. 玩家选前能看到什么承诺、成本和风险语义。
6. `branchImpact.title` 是否是可命名主题。
7. `branchImpact.detail` 是否不是系统说明句。
8. 本次是否会提升或缓和失败压力。
9. 是否通过伪分支检查。

## 10. 内容扩写的设计底线

- 不允许只扩文本，不扩卷内承接结构。
- 不允许只扩隐藏状态，不扩读取点。
- 不允许把卷末收束完全让给支线。
- 不允许把关键失败写成“作者心里知道、规则书里没写”的隐规则。
- 不允许让章节作者依赖代码细节才能判断内容是否合格。
