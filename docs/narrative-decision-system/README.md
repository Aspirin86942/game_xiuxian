# 理想化叙事决策系统规则书 v2

## 1. 目的

本目录用于定义《灵光修仙传》叙事决策系统的理想目标态，而不是继续把当前代码行为逐段展开成事实手册。

本册采用三层写法：

- `目标态规范`：描述未来应当实现的理想叙事决策闭环。
- `现状基线`：只在 `01-current-behavior.md` 中记录当前实现、关键偏差与落地约束。
- `落地差距`：说明从当前实现走向目标态时，哪些语义必须被迁移、哪些旧约束可以被放弃。

默认原则：

> 先改规则书，再改代码。

补充说明：

- 运行时真实行为，当前仍以代码与 `01-current-behavior.md` 为准。
- 未来改代码时，目标态设计、参数、契约与内容接口，以本册其他文件为准。

## 2. 使用顺序

1. 先看 `00-design-principles.md`，确认理想化叙事决策系统到底优先优化什么。
2. 再看 `01-current-behavior.md`，确认当前实现离目标态还差什么、哪些差距不能忽视。
3. 再看 `02-parameters-and-formulas.md`，确认目标态参数、风险档位、分支影响规格和观测指标。
4. 再看 `03-implementation-contract.md`，确认未来实施时哪些语义、顺序、可见性和测试门槛必须保住。
5. 再看 `04-change-worklog.md`，确认规则书版本切换、前提变化与回退条件。
6. 最后看 `05-content-authoring-interface.md` 与两个附录，确认作者如何接入章节、choice、分支影响、隐藏承接与终局种子。

## 3. 工作流

1. 先在 `00` 明确本次改动是否符合理想目标态，而不是先看“代码里现在怎么写”。
2. 再在 `01` 找当前实现的最小基线和差距，判断这次是文档先行、代码跟进，还是仅补记差距。
3. 再在 `02` 确认受影响的参数组、观测指标和联动项。
4. 再在 `03` 判定改动属于 A/B/C 哪一级，是否触发额外测试和审计。
5. 改完规则或改完代码后，在 `04` 留下记录，不把设计判断只留在聊天记录里。
6. 若新增剧情资产，回到 `05` 与附录，核对决策承诺、分支影响、隐藏承接读取点与伪分支自检。

## 4. 目录职责

- `00-design-principles.md`：理想化设计宪章，回答“好 choice 应该长什么样、为什么”。
- `01-current-behavior.md`：最小现状基线，只记录当前实现、关键偏差与落地约束。
- `02-parameters-and-formulas.md`：目标态参数手册，定义风险档位、分支影响规格、隐藏承接窗口与观测指标。
- `03-implementation-contract.md`：目标态实施契约，定义稳定接口、流程顺序、UI 揭示与测试门槛。
- `04-change-worklog.md`：规则级变更记录，记录前提改变、兼容影响与回退条件。
- `05-content-authoring-interface.md`：理想化作者接口，规定章节、choice、分支影响、隐藏承接与终局种子如何接入系统。
- `appendix-a-state-matrix.md`：状态矩阵、可见性矩阵、后果矩阵与单次选择生命周期。
- `appendix-b-test-scenarios.md`：设计审计、目标态验收场景与迁移验证入口。

## 6. 本册定位

- 本册是当前系列中的唯一 mature 分册。
- 本册不是“当前代码全文说明书”，而是“理想化叙事决策系统 v2 的目标规则书”。
- 本册保留 `01-current-behavior.md` 只是为了记录现状基线与落地差距，不代表其余文件要被当前代码形状绑死。

## 7. 本册负责什么

- 定义理想的决策输入层：玩家在做选择前必须知道什么、允许不知道什么。
- 定义理想的可见后果层：选择后应如何给出单条分支影响，并让失败压力独立可见。
- 定义理想的隐藏承接层：哪些后果可以延迟到账，但不能继续以可见多卡直接堆叠。
- 定义理想的分支记忆层：哪些状态才值得长期保存，哪些“只写不读”的痕迹不应进入系统。
- 定义理想的揭示契约：哪些信息必须选前揭示、哪些只能选后解释、哪些允许延迟到账。
- 定义理想的内容接入契约：作者新增章节、choice、分支影响与失败路径时最低需要满足什么。

## 8. 本册不负责什么

- 单句台词润色或章节全文备份。
- 战斗数值、挂机收益、掉落概率等非叙事决策主体系统。
- 仅靠审美判断的 UI 微调。
- 不带系统状态写入与后续读取的纯文案扩写。

## 9. 与其他分册的交界

- 若未来拆出 `branch-memory-architecture`，更细的状态命名、长期记忆维度与归档策略可迁出。
- 若未来拆出 `echo-and-reveal-system`，更细的分支影响命名规则、隐藏承接兑现方式与去重策略可迁出。
- 若未来拆出 `failure-pressure-model`，更细的失败压力、濒危区间与终局种子规则可迁出。
- 即便未来拆册，本册仍保留系统级地图，负责说明三类能力如何组成完整叙事决策闭环。

- 理想化叙事决策系统 v2 的一句话目标是：每次选择都应同时构成“可读的决策承诺、可回看的单条分支影响、可管理的失败压力、可追踪的隐藏承接与终局回指”。

## 12. 维护规则

- 任何对目标态的修改，至少同步检查 `00`、`02`、`03`、`05` 是否仍一致。
- 任何把当前实现迁向目标态的实施，必须同步更新 `01` 和 `04`，避免差距失真。
- 任何新增章节、choice、分支影响或终局，若不能通过 `05` 与附录中的伪分支和承接检查，就不应接入系统。
- 若未来继续升级到 v3，应先在 `04-change-worklog.md` 记录为什么 v2 不足，再重写，不在聊天中隐式漂移。
