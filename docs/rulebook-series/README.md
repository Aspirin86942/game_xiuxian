# 规则书系列索引

## 1. 系列目标

- 用一组规则书持续维护《灵光修仙传》的叙事玩法，但把“理想目标态”与“当前实现”明确拆开。
- 当前第一本成熟分册仍是 `narrative-decision-system`，但其职责已升级为：定义理想化叙事决策系统 v2 的目标规格、实施契约与内容接入方法。
- 当前代码真实行为不再由系列索引承担说明；系列索引只负责导航、成熟度、边界与同步规则。

## 2. 定位

- 本索引只负责导航、分册关系、成熟度说明与维护规则。
- 真正的目标态规范写在成熟分册内；真正的当前实现差距写在分册自己的 `01-current-behavior.md`。
- 当前系列目录：`rulebook-series`
- 当前唯一 mature 分册：`narrative-decision-system`

## 3. 当前已成熟分册

- `narrative-decision-system`：当前唯一 mature 分册，负责理想化叙事决策系统 v2 的目标设计与实施约束。

## 6. 阅读顺序

1. 先看 `narrative-decision-system/README.md`，确认该分册的三层语义：目标态规范、现状基线、落地差距。
2. 再按 `00 -> 01 -> 02 -> 03 -> 04 -> 05` 的顺序阅读成熟分册。
3. 最后看附录，核对状态矩阵、可见性矩阵、后果矩阵、设计审计和目标态验收场景。
4. 回到本索引，核对分册边界、成熟度与同步规则。

## 7. 维护规则

- 先改规则书，再改代码。
- 系列索引不保存系统事实、参数表或实施契约；这些内容必须落在成熟分册内。
- 当目标态规则与当前代码冲突时，必须先在对应分册的 `01-current-behavior.md` 记录差距，再决定是否实施。
- 新分册未成熟前，只能登记为计划中分册，不能用空壳目录冒充完成。

## 8. 同步规则

- 分册目录、定位、成熟度变化时，必须同步更新本索引。
- 若成熟分册从“现状事实书”升级为“目标态规则书”，必须在索引中显式更新定位，而不是沿用旧说明。
- 若未来把 `narrative-decision-system` 的部分能力拆给新分册，索引中必须同步写清新的边界和主责。
