# 00. 设计原则

## 1. 目的

用一组明确原则约束地点委托系统的后续设计与实现判断，避免委托继续变成“换个标题的主线尾巴”。

## 2. 系统优化优先级

### 2.1 第一优先：委托服务地点生态

- 委托首先要回答“这件事为什么只能发生在这个地点”，而不是“这一卷还有哪笔旧账没补”。
- 青牛镇、太南山、黄枫谷、乱星海必须分别形成不同的委托气质。
- 若“再加一条委托”和“保持地点辨识度”冲突，优先保地点辨识度。

### 2.2 第二优先：生命周期简单可解释

- 玩家要知道这笔委托为什么会在这里出现。
- 玩家要知道它为什么能接、为什么会失败、为什么做完就收住。

### 2.3 第三优先：不碰主线收束职责

- 委托不负责卷末收账。
- 委托不负责替主线章节补解释。
- 委托可以共享地点与世界资产，但不能共享主线的收束义务。

## 3. 什么叫好地点委托

一个好的地点委托至少满足：

1. 它能被一句话说明为什么只能发生在这个地点。
2. 玩家知道它为什么在当前境界区间出现。
3. 玩家知道自己是在跑差使、做买卖、探路还是押货。
4. 它至少改变两层系统痕迹，例如资源 + 路线倾向，或旗标 + 结果摘要。
5. 它办完后能自然收住，不需要主线替它补尾。

## 3.1 什么叫好抉择

- 玩家知道自己是在求稳、求利、求名声还是求后手。
- 抉择至少改变两层系统痕迹，而不是只有奖励数字差异。
- 抉择要帮助地点更鲜明，而不是把主线包袱重新挂回来。

## 4. 什么叫坏地点委托

- 换个地点名也完全说得通，缺少地点辨识度。
- 只给灵石或道具，不写地方风气、路线倾向或结果留痕。
- 把主线旧账换个标题塞进委托面板。
- 把一次性委托写成可重复刷取的赏金循环。
- 让委托结果决定主线是否能解释完整。

## 4.1 什么叫坏抉择

- 文案不同，但资源、旗标、路线和结果摘要完全相同。
- 只有收益差异，没有风险语义差异。
- 玩家做完后，仍然说不清这笔委托到底留下了什么。

## 5. 合法的收束方式

地点委托在当前系统里只允许：

- 进入 `available` 后由玩家接取。
- 进入 `active` 后通过一次 choice 结算成 `completed` 或 `failed`。
- 结算后只作为当前地点的结果留痕展示，不跨地点悬挂。

不允许的做法：

- 继续保持跨地点可见的 `active`
- 靠主线章节补出委托结局
- 靠作者记忆“以后再补”

## 6. 可接受的不透明

- 可以不在选前显示精确奖励数值，但必须让玩家知道偏稳、偏利还是偏名声。
- 可以不公开全部隐藏旗标，但必须让玩家知道这笔委托大致会留下哪类痕迹。
- 可以保留地方风闻，但不能把“为什么会出现 / 为什么不能接”全部藏起来。

## 7. 高风险

以下改动默认高风险：

- 改变委托状态枚举、开放条件或单活跃限制
- 让多个地点的委托在同一面板里交叉污染
- 把一次性奖励设计成可重复刷取
- 把不适配当前项目口径的原著桥段直接导入委托池
- 用新字段写入正式存档却不补兼容层

## 8. 设计判断捷径

未来判断一笔地点委托是否值得落地，优先问六个问题：

1. 它为什么只能发生在这个地点。
2. 它为什么在这个境界段才会出现。
3. 它偏凡俗差使、山市买卖、宗门杂务还是海上生计。
4. 玩家为什么会接、会犹豫、会失手。
5. 它会留下哪些轻量痕迹。
6. 如果删掉它，这个地点会不会立刻变得更空、更同质化。

## 9. 系统一致性

- 地点、境界门槛、UI 词面与存档状态必须互相对得上。
- 已经完成或失败的委托，不应在别的地点继续以未收束状态出现。
- 主线章节可以共享地点素材，但不应依赖委托结果完成自己的解释闭环。
