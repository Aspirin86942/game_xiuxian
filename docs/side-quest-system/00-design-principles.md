# 00. 设计原则

## 1. 目的

用一组明确原则约束支线任务系统的后续设计与实现判断，避免支线变成“只有标题和文案的装饰面板”，却没有真正的状态承接、奖励语义和冲突处理。

## 2. 系统优化优先级

1. 先保证支线与主线、NPC、资源系统之间的语义一致，再谈扩容速度。
2. 先保证玩家能理解“为什么这条支线会出现/消失/失败”，再谈任务数量和层层嵌套。
3. 先保证作者能按统一接口稳定扩写，再谈个别任务的戏剧强度。

## 3. 什么叫好抉择

- 一个好的支线 choice，必须让玩家知道自己是在补旧债、护旧情、换利益、试风向，还是干脆切断牵连。
- 一个好的支线 choice，至少同时改变两层系统痕迹，例如关系变化 + 后续读取点，或资源收益 + 后续线索。
- 一个好的支线任务，即便不推进主线，也能让玩家感觉它确实重新定义了“你和谁站在一边、你愿不愿意付代价、别人之后会怎么记你”。

## 4. 什么叫坏抉择

- 只换按钮文案，不换触发条件、奖励结构、后续读取点。
- 只给灵石或道具，不写关系后果、名声后果或后续回响。
- 把原著桥段直接改写成插曲文本，却不适配当前项目的韩立式克制口径。
- 让支线直接抢断主线播放，或让主线推进后支线 silently 丢失却没有任何交代。

## 5. 可接受的不透明

- 可以不在选前显示精确奖励数值，但必须让玩家理解是偏“人情”、偏“资源”、偏“护住退路”还是偏“先拿收益”。
- 可以不在选前公开未来具体章节读取点，但必须让玩家知道该任务会留下后续影响。
- 可以保留隐藏承接，但不能把失败/错过条件完全藏起来，导致玩家不知道自己为什么再也接不到。

## 6. 高风险

- 直接改变任务状态流转的枚举、语义或优先级。
- 让支线在没有冲突策略的前提下批量出现，造成任务面板噪声。
- 把唯一奖励设计成可重复刷取。
- 把原著中不适配当前项目的露骨、暴力或爽点桥段直接导入任务池。
- 用新字段写入正式存档，却不补兼容与默认值策略。

## 7. 系统一致性

- 支线的入口必须和当前项目的“线索先出现、后续再被认出来”口径一致，不突兀切成现代任务看板。
- 支线与主线的交界必须清楚：主线优先推进，支线承担补账、补人情、补后果与扩大世界厚度。
- 奖励强度必须和章节窗口、境界阶段、资源紧缺程度挂钩，不能早期给过重奖励，也不能后期只给空文案。
- 原著只能提供剧情骨架，最终文本必须服从本项目现有叙事口径与系统边界。
