# 05. 内容接入接口

## 1. 目的

本文件定义支线任务系统的内容资产接入规则，确保任务原型、任务章节、任务 choice、奖励包和后续读取点都能以统一方式接入系统。

本文件同时规定原著参考素材的改写边界，避免把离线文本直接搬进项目正文。

## 2. 一个任务最少要定义什么

一个合格的支线任务至少要定义：

- `任务标题`
- `任务分类`
- `volumeAnchor`
- `触发条件`
- `接取方式`
- `成功条件`
- `失败或错过条件`
- `奖励包`
- `closureMode`
- `followupHook`

每条任务还必须说明：

- 它主要在补哪种旧账：旧案、旧情、师门、人情、利益还是退路。
- 它和哪条主线窗口产生交界。
- 它会写入哪些旗标、关系、路线或后续回响。

## 2.1 玩家可见词面规则

支线系统内部可以继续使用 `side quest`、`formal quest`、`legacy clue`、状态枚举等术语；但玩家可见层必须统一改写为世界观词面。

默认映射如下：

| 内部术语 / 语义 | 玩家可见词面 | 典型位置 |
| --- | --- | --- |
| `side quest` / 支线任务 | `旧事` | 旧事区标题、条目摘要 |
| legacy clue | `旧事线索` | 旧线索标签 |
| `available` | `可应旧事` | 条目状态标签 |
| accept action | `应下这桩旧事` | 接取按钮 |
| blocked by other active quest | `另有旧事未了` | 阻断按钮文案 |
| empty state | `暂无旧事上门` | 旧事区空态 |

补充要求：

- 文案口径采用“半世界观化”，既不写系统后台说明，也不硬拗成难读的古白话。
- 若要表达奖励、关系、成本或错过，优先写成旧债、人情、门墙、风声、去路与代价，而不是抽象成产品术语。
- 参考《凡人修仙传》时，只借母题与气质，不直接复制原句。

## 3. 一个合格 choice 至少要有哪三层差异

### 3.1 表层差异：任务承诺

- 玩家必须能理解这次是在补债、救人、封口、取证、换利益还是切断关系。

### 3.2 中层差异：任务成本

- 玩家至少要看到一种代价，例如灵石、人情、时间窗口、关系波动或错过风险。

### 3.3 深层差异：任务后果

- choice 至少命中以下三项中的两项：
  - 一项奖励包变化
  - 一项关系 / 旗标 / 路线变化
  - 一项 `followupHook`

如果只有文案差异，没有任务后果差异，默认视为伪分支。

补充要求：

- 玩家可见结果摘要不能直接写成“资源兑换”“关系后果”“情绪奖励”。
- 若要告诉玩家这桩旧事留下了什么，必须写成世界里的结果：是认下旧债、背走名声、欠下人情、断开去路，还是把祸根暂时按住。

## 4. 何时必须写显式 `tradeoff`

- 任务是唯一奖励或限时任务时
- 任务会让玩家在旧情 / 旧债和资源 / 安全之间明确取舍时
- 任务会推动关系大幅变化时
- 任务失败或错过会改变后续线索池时
- 任务在卷末需要被主线接管解释时

## 4.1 何时只允许走默认 `tradeoff`

- 低风险回信、常规探路、轻量补账这类不改变卷末结构的位置。
- 只是补一层关系温度，不改变结果态类型的选择。
- 不涉及唯一奖励、不涉及跨卷读取点变化的普通分支。

## 5. 合法的 `closureMode`

- `volume_close`
  - 本卷内结清，不再跨卷。
- `seed_forward`
  - 本卷结算，但保留低权重读取点。
- `convert_to_main`
  - 卷末由主线章节回收解释。

作者不允许自行发明第四种卷末去向。

## 6. 第一卷支线原型默认

改编第一卷《七玄门风云》优先围绕以下原型展开：

| 任务原型 | 任务语义 | 推荐 `closureMode` | 当前素材来源 |
| --- | --- | --- | --- |
| 墨府旧账 | 处理墨府余波、补活人账 | `volume_close` | `旧药账`、旧 `8 墨府旧事` |
| 药童残影 | 清神手谷旧案、决定是否继续翻旧账 | `convert_to_main` | `药童残影`、旧 `9 曲魂初现` |
| 门内旧友 | 处理厉飞雨和门内人情 | `volume_close` | 旧 `3 厉飞雨之疾` 的关系余波 |
| 门内守望 | 卷末守门或切割立场 | `volume_close` 或 `seed_forward` | 第一卷第 7 章卷末清账窗口 |

## 7. 原著参考文本使用边界

- 离线参考素材路径：
  - `D:\Program_python\game_xiuxian\Fan Ren Xiu Xian Chuan [ Geng Xin Zhi  Di - Wang Yu.txt`
- 该文件只用于：
  - 校对人物关系骨架
  - 校对阶段性事件顺序
  - 提炼任务母题，例如旧债、旧案、旧情、守门、断尾
- 该文件不得用于：
  - 直接复制原文段落
  - 直接搬运露骨、敏感或不适配当前项目口径的桥段
  - 直接作为代码、测试脚本或配置表的数据源

## 8. 作者最小接入清单

1. 这条任务属于哪一卷。
2. 它补的是哪类卷主题。
3. 它为什么会在这个主线窗口出现。
4. 它的失败或错过条件是什么。
5. 它的奖励包是什么。
6. 它会写入哪些关系、旗标、路线或 `followupHook`。
7. 它在卷末怎么收。
8. 它是否通过伪分支检查。
9. 它的玩家可见词面是否已按“旧事 / 旧事线索 / 可应旧事 / 应下这桩旧事”落地。

## 9. 设计底线

- 不允许只扩文本，不扩任务后果。
- 不允许只扩隐藏状态，不扩 `followupHook`。
- 不允许把明明应卷内解释的旧账全部拖到下一卷。
- 不允许继续让第一卷卷末挂满未解释的旧支线。
- 不允许把内部任务术语直接塞给玩家当界面标题或按钮文案。

### 5.1 主线回收旧事时的晚期口径

- 若任务结果由主线卷末或终局前章节回收解释，主线正文必须直接落到旧债、旧人、门墙压力或具体场景上。
- 不允许用“这条线没断，只是以后再说”式桥接句把应卷内解释的结果继续往后拖。
