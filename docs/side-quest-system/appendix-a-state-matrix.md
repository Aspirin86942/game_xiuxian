# 附录 A. 状态矩阵

## 1. 目的

本附录收纳支线任务系统从“当前显性线索层”走向“正式任务状态机”时的最小状态矩阵，避免实现阶段重复争论状态名和转换边界。

## 2. 当前线索层矩阵

| 当前状态 | 当前是否存在 | 当前可见性 | 当前写入存档 | 当前可交互 |
| --- | --- | --- | --- | --- |
| 无线索 | 是 | 显示占位文案 | 否 | 否 |
| 有线索 | 是 | 显示标题 + 正文 | 否 | 否 |
| 可接取 | 否 | 无 | 否 | 否 |
| 进行中 | 否 | 无 | 否 | 否 |
| 已完成 | 否 | 无 | 否 | 否 |
| 已失败 | 否 | 无 | 否 | 否 |
| 已错过 | 否 | 无 | 否 | 否 |

## 3. 未来正式任务状态矩阵

| 状态 | 进入条件 | 玩家可见信息 | 允许迁移到 | 备注 |
| --- | --- | --- | --- | --- |
| `locked` | 未满足触发条件 | 可隐藏或只显示模糊线索 | `available` | 默认初始态 |
| `available` | 满足触发条件 | 标题、分类、条件摘要、奖励概览、是否限时 | `active` / `missed` | 可接取但未接 |
| `active` | 玩家接取或隐式激活 | 当前状态、剩余窗口、进度提示 | `completed` / `failed` / `missed` | 正式进行中 |
| `completed` | 达成成功条件 | 完成结果、奖励已结算、后续读取提示 | 终态 | 唯一奖励只能结算一次 |
| `failed` | 命中失败条件 | 失败原因、未获得或部分获得奖励 | 终态 | 失败不应伪装成未触发 |
| `missed` | 超过时窗且未完成 | 错过原因、后续是否还有替代线 | 终态 | 与 `failed` 语义不同 |

## 4. 冲突矩阵

| 场景 | 推荐裁决 | 原因 |
| --- | --- | --- |
| 主线推进与支线触发同回合命中 | 主线优先，支线保留为线索或 `available` | 不抢断主线播放 |
| 同一 `exclusiveGroup` 命中多个任务 | 只保留优先级最高者 | 避免互斥任务同时可接 |
| 同阶段普通支线过多 | 限制显性可见数量 | 保持 375x667 单屏可读性 |
| 唯一奖励任务重复命中 | 仅首次可结算 | 防刷取 |

## 5. 当前与目标态差距

- 当前只有“线索是否显示”这一层状态，没有正式任务生命周期。
- 当前线索不写入专门存档，目标态必须补任务状态持久化。
- 当前 UI 只展示标题和正文，目标态至少需要补状态、奖励概览和失败/错过提示。
