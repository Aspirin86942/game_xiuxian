# 《灵光修仙传》第二轮扩充试玩记录

## 方式说明

- 本轮已完成脚本式完整路径试玩与代码走查。
- 当前环境未找到可直接调用的本地浏览器命令，无法完成真正的屏幕级手工试玩或截图验证。
- 因此本记录分为：
  - 已验证：通过 `tests/story-smoke.js` 实际跑通的路径与状态流。
  - 待人工实机补验：手机视口下的首屏密度、模态遮挡、滚动手感。

## 已验证路径

### 正道救援线

- 路径：救厉飞雨 -> 收证据 -> 稳妥反杀墨大夫 -> 护墨府 -> 救南宫婉 -> 守宗门 -> 飞升
- 结果：
  - 主线可从第 0 章推进到结局。
  - 小境界事件会穿插触发，但不会抢断当前主线播放。
  - `storyCursor.source` 能在 `main -> level -> main -> ending` 间切换。
  - 南宫婉关系与正道分值会持续累积，终局倾向正确。

### 魔路夺宝线

- 路径：抢先动手 -> 烧遗书 -> 取宝离城 -> 禁地杀人夺宝 -> 投向魔道 -> 单人逃生 -> 留在人界
- 结果：
  - 分支可持续影响中后期文案与 `getEchoes()` 输出。
  - 魔路倾向能在终局与中后期提示里体现，不再只是单章反馈。
  - 资源扣减与掉落不会把库存打成负数。

### 苟修藏锋线

- 路径：独行开局 -> 埋尸保密 -> 许诺墨彩环 -> 买消息 -> 慢修禁地 -> 假参战 -> 藏锋到终局
- 结果：
  - 苟修路线可稳定走通至结局。
  - “避因果、留退路”的回响会出现在后期剧情和 echo 摘要里。
  - 带资源成本的小境界事件选项会正确禁用。

## 小境界事件验证

- 已验证 15 个小境界事件全部存在且可触发。
- 每个小境界事件完成后，`levelStoryState.events[eventId].completed = true`。
- 每个小境界事件都能回到主线调度，不会卡死在事件层。

## 已修复或确认通过的风险

- 主线不会因小境界事件插入而中断。
- `storyCursor` 已区分 `main / level / ending`。
- 资源不足时无法选择带成本的主线或等级事件选项。
- 背包库存不会出现负数。
- 跳过本段、逐句推进、进入抉择三者的切换逻辑已通过脚本验证。
- 终境继续点击突破不会把角色错误打回低境界。

## 待人工实机补验

- `360x780 / 375x667 / 390x844` 下首屏是否确实比上一版紧凑。
- 剧情页历史句区域是否仍会在某些长文本下把选项挤出首屏。
- 背包、设置、对话、战斗模态在手机屏上的滚动手感和关闭区域是否自然。
- 底部页签是否足够适合单手拇指点击。

## 当前结论

- 脚本式完整路径试玩通过，状态机与剧情调度没有发现阻塞级问题。
- 真正剩下的主要风险集中在手机端视觉密度与交互手感，需要后续在真实浏览器或设备上补一轮人工验证。
